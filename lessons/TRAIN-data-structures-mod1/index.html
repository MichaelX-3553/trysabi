    Section 1

    What Is a Data Type?

    Before a computer can work with any information, it needs to know what kind of thing it is dealing with.

    🌾 Think About Your Farm

    On your farm, you have different kinds of things: yams, maize, cassava. You do not store them the same way. Yams go in a barn. Maize hangs to dry. Cassava goes in sacks. Each kind of thing needs its own way of storing.

    A computer works the same way. It stores numbers, letters, and yes/no answers. Each kind is called a data type.

    A data type tells the computer two things:

    • What values are allowed — like whole numbers, or letters, or true/false
    • What you can do with it — like add numbers or join words
    Think about a player's jersey. The number on the back (like 10) is one kind — a number. The name ("Musa") is another kind — text. You cannot add two names like numbers. Each kind follows different rules.

    The 4 Simple Data Types

    These are the basic building blocks. Every computer program uses them.

    🔤

    Character

    A single letter, digit, or symbol.

    Examples: 'A' '5' '@'

    Like one seed in your hand.
    🔢

    Integer

    A whole number — no decimal point.

    Examples: 7 -3 100

    A jersey number: 10, not 10.5
    📐

    Real (Float)

    A number with a decimal point.

    Examples: 3.14 -0.5

    Measuring 2.5 kg of yams on a scale.
    ✅

    Boolean

    Only two values: true or false.

    Like a yes/no answer.

    "Was it a goal?" — Yes or No.

    Section 2

    Abstract Data Types

    A plan for how to store and use data — before you build anything.

    🏗️ The Blueprint Idea

    Before you build a yam barn, you have a plan in your head: "It will hold yams. I can add yams, take yams out, and count how many I have." This plan is not the barn itself — it is the idea of the barn.

    An Abstract Data Type (ADT) is like that plan. It describes:

    • What kind of data it holds
    • What actions you can do with that data

    It does NOT say HOW to build it. Just WHAT it should do.

    A "team lineup" is an abstract idea. It says: "11 players, each in a position." But HOW you write it — on paper, on a whiteboard, on a phone — those are different ways to build the same idea.
    💡
    Common ADTs: Arrays, Lists, Queues, Stacks, Trees. You will learn about these soon!

    Section 3

    Key Terms You Must Know

    Learn the words computer people use to talk about organized data.

    📦 What Is a Data Structure?

    A data structure is a way to store and organize data so you can find, add, or remove things quickly.

    You have different ways to organize yams: in a pile (stack), in a line at the market (queue), or in numbered rows of mounds (array). Each way has its own rules. Each is a data structure.

    An algorithm is a set of step-by-step instructions to solve a problem. Like a recipe for cooking.

    Your coach says: "Pass to the defender, then to the midfielder, then shoot." Those steps in order are an algorithm for scoring a goal.

    Section 4

    How Data Structures Are Grouped

    Data structures come in families. Here is the family tree.

    Two Big Families

    1. Primitive (Simple)

    Built into every computer. Cannot be broken into smaller parts.

    Integer, Float, Character, Boolean

    Like raw seeds — the most basic unit before planting.

    2. Non-Primitive (Complex)

    Built from simple types. Organized in special ways.

    Arrays, Lists, Stacks, Queues, Trees, Graphs

    Like a full team — built from individual players.

    Linear vs Non-Linear

    Linear (in a line)

    Data is one after another, like a chain.

    Array, Linked List, Stack, Queue

    Yam mounds in a straight row.

    Non-Linear (branching)

    Data branches out like a tree.

    Tree, Graph

    A tournament bracket — branching paths.

    Static vs Dynamic

    Static = size is fixed. Once set, it cannot grow or shrink.

    A barn with exactly 20 shelves. You cannot add a 21st shelf.

    Dynamic = size can change while the program runs.

    A bench of substitutes — players can join or leave during the match.

    Section 5

    Meet the Array

    The most common way to store a list of items in a computer.

    🌾 A Row of Numbered Yam Mounds

    Mathias, picture this: You make a straight row of 10 yam mounds. You number each mound: 0, 1, 2, 3... up to 9. Each mound holds exactly one yam. If you want the yam in mound 5, you go straight there — no need to check 0, 1, 2, 3, 4 first.

    An array works like this row. Key facts:

    • Each item has a position number called an index
    • The index starts at 0, not 1
    • All items must be the same type
    • The array has a fixed size
    A team lineup card with 11 slots. Slot 0 is the goalkeeper, slot 1 the first defender, and so on. Each slot holds one player name. That card is an array!

    Creating an Array in Code

    // Say you want an array of whole numbers int[] myArray; // Set the size to 10 spaces myArray = new int[10]; // Or create with values directly: int[] ages = {18, 20, 22, 19, 21, 23}; // Get value at position 2: int x = ages[2]; // x is now 22 // Find how many items: int len = ages.length; // len is 6
    ☝️
    The first item is at index 0. In an array of 6 items, the last index is 5.

    Section 7

    Multi-Dimensional Arrays

    Arrays inside arrays — like a table with rows and columns.

    From a Row to a Grid

    Picture your farm field: You have 3 rows of mounds, each row with 4 mounds. That is a grid — 3 rows × 4 columns = 12 mounds. To find one, you say: "Row 1, Column 2."

    A 2D array works the same way. You need two numbers to find an item: the row and the column.

    // Java: a 3×4 grid int[][] grid = new int[3][4]; // Python: a 3×3 grid with values grid = [[1,2,3], [4,5,6], [7,8,9]] // Get value at row 0, column 1: grid[0][1] // gives 2
    A league table: each row is a team, each column is a stat (wins, draws, losses). To find Team 2's wins: table[2][0].

    • Math work — tables, grids, charts
    • Screen displays — lists of items users see
