Section 1

How Computers Find Things

There are two ways to find something: jump straight to it, or search from the start.

Two Ways to Find Data

Think about this, Mathias. You have two ways to find a yam:

🌾 Random Access — Like a yam barn with labels: Every yam has a spot number. You want yam #50? You walk straight to spot 50. It does not matter if the barn has 10 yams or 10,000 — you go right to the spot. The position does not change how fast you find it.
⚽ Sequential Access — Like a single-file line: Imagine players standing in a line. To find player #7, you must check player #1, then #2, then #3... all the way to #7. Finding someone at the end takes much longer than finding someone at the start.

Random Access means the position does not change the speed. Your computer's memory (RAM) works this way — it can grab any piece of data instantly.

Sequential Access means you must go through items one by one from the start. Old tape drives and text files work this way.

Arrays — Your Numbered Shelf

An array is like a shelf with numbered slots. Each slot holds one item.

🌾 Think of a yam barn with 100 numbered spaces. Space 1 holds one yam, space 2 holds another. You can put a yam in space 50 just as fast as space 1. That is random access.

In code, it looks like this:

names[1] = "Alex" // Put "Alex" in slot 1 names[1000000] = "Zeke" // Put "Zeke" in slot 1000000 // Both take the same time!

Section 2

Stacks — Last In, First Out

Like a pile of yams — the last one you put on top is the first one you take off.

What is a Stack?

LIFO stands for Last In, First Out. The last item you add is the first item you remove.

🌾 Stacking yams in a basket: You put yams in one by one. The last yam you placed sits on top. When you take yams out, you take the top one first. You cannot grab a yam from the bottom without removing the ones above it.
⚽ Stacking jerseys after training: Players throw jerseys in a pile. The last jersey thrown on top is the first one picked up. The first jersey thrown is buried at the bottom.

📝 Stack Code

int top = 0; String[] stack = new String[100]; public void push(String newItem) { top = top + 1; // Move up one spot stack[top] = newItem; // Put the item there } public String pop() { String item = stack[top]; // Take the top item top = top - 1; // Move down one spot return item; }
💡The top variable keeps track of where the top of the stack is. Push moves it up, Pop moves it down.

Section 3

Queues — First In, First Out

Like a line at the grinding machine — the first person in line is served first.

What is a Queue?

FIFO stands for First In, First Out. The first item you add is the first item you remove.

🌾 Waiting at the grinding machine: Farmers line up. The first farmer who arrives gets served first. New farmers join at the back. This is fair — everyone waits their turn.
⚽ Penalty shootout order: Players line up to take penalty kicks. The first in line kicks first. A new player joins at the back of the line.

Deque — The Double-Ended Queue

A deque (say "deck") works like both a stack AND a queue. You can add or remove from both ends.

⚽ Think of substitutions: Normally players wait in line (queue). But if the coach says "you go in NOW!" — that player jumps to the front. A deque allows pushing to the front for urgent items.

This is useful for priorities. Some jobs are more urgent. An urgent job can be pushed to the front.

💡Stacks, Queues, and Deques are called Abstract Data Types (ADT). This means they are general ideas — you can build them in many different ways.

Section 5

Trees — Family Structure

A tree stores data like a family tree — with parents, children, and branches.

What is a Tree?

Mathias, a tree in computing is NOT a straight line. It branches out, like a real tree — or like a family.

🌾 Like a farming family: The grandfather (root) has children. Each child has their own children. The family branches out. Everyone connects back up to the grandfather.

A tree has nodes (each box that holds data) connected by edges (the lines between them).

Key Tree Words

WordMeaningAnalogy
RootThe top node. Only one.🌾 The grandfather of the farm family
ParentA node with children below it⚽ The coach above the captain
ChildA node below a parent🌾 Sons/daughters of a farmer
LeafA node with NO children⚽ A player with nobody below them

Types of Trees

1. General Tree: Each node can have ANY number of children. No limit.

🌾 Like a farmer who can have 1, 3, or 10 children. No rule about how many.

2. Binary Tree: Each node can have AT MOST 2 children — a left child and a right child.

⚽ Like each player can only pass to TWO people — one on the left, one on the right.

3. Binary Search Tree (BST): A binary tree with a sorting rule — smaller values go LEFT, bigger values go RIGHT.

🌾 Imagine sorting yams by weight. Lighter yams go in the left basket, heavier ones go right. At each step, you sort again. This makes finding any yam very fast.

Section 6

Binary Search Trees

A special tree where smaller goes left, bigger goes right. This makes searching fast!

The BST Rule

Mathias, this is the most important part about BSTs:

  • Values smaller than the node → go to the LEFT
  • Values bigger than the node → go to the RIGHT
🌾 Sorting yams by size: You pick up a yam. Is it lighter than the one in your hand? Put it LEFT. Heavier? Put it RIGHT. Do this at every step. Now to find any yam, just keep asking "lighter or heavier?" — you find it very fast!

Search in a BST

To find a value, start at the root. If the value is smaller, go left. If bigger, go right. Repeat until you find it or reach the end.

Insert in a BST

To add a new value, follow the same path (left if smaller, right if bigger). When you reach an empty spot, put it there.

Advantages & Disadvantages

✅ Fast searching — at each step you cut out half the tree.

✅ Easy to search — just compare: smaller or bigger?

❌ Can become unbalanced — if you add sorted numbers (1,2,3,4,5...), the tree becomes a straight line. This is called skewed.

⚽ A skewed tree is like all players standing in a single line — you lose the advantage of branching. A balanced tree keeps both sides roughly equal, so searching stays fast.

Section 7

Tree Traversal

Traversal means visiting every node. There are 3 ways to do it.

Three Ways to Visit Every Node

Traversal means going through every node in the tree, one by one. There are 3 orders:

1. In-Order: Left → Root → Right

🌾 Check the left field first. Then check the boss. Then check the right field. The boss is IN the middle.

2. Pre-Order: Root → Left → Right

⚽ The coach checks HIMSELF first (PRE = before). Then the left side. Then the right side.

3. Post-Order: Left → Right → Root

🌾 Check all workers first. The boss checks himself LAST (POST = after).
