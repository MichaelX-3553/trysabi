Section 1

Arrays: Numbered Boxes in a Row

Think of your yam farm. You make heaps in a straight line. Each heap has a position. That is exactly what an array is for a computer.

🌾 What Is an Array?

An array is a straight line of boxes. Each box has a number (called index). Each box holds one piece of data. Computer counts from zero, not one.

🌾 Farming Analogy

Your farm has 9 yam heaps in a row. Heap 0, Heap 1, Heap 2... all the way to Heap 8. Each heap has yams inside. You can walk to any heap and check how many yams are there.

💻 How to Write It in Code

In Java programming language, this is how you create and use arrays:

Java
// Create 10 empty boxes int[] heaps; heaps = new int[10]; // Create boxes and put values inside int[] values = {2, 4, 5, 8, 11, 3, 23, 33, 12}; // Get the item at position 5 int item = values[5]; // gives you 3 // Get total number of boxes int size = values.length; // gives you 9
💡

int[] means "a row of numbers." new int[10] means "make 10 boxes." values[5] means "give me what is inside box number 5."

Section 2

Access Methods: How Do You Reach Your Stuff?

Access method means: how does the computer get to its data? Can it jump to any position, or must it go one by one? There are different ways.

🎯 Random Access

Jump directly to any position. The location does not slow you down. Position 1 and position 1000 are equally fast to reach.

On your farm, you can walk directly to any heap. Heap 3 or Heap 50 — you just go there. No need to pass through every heap before it.

Read Write Jump

Examples: Hard disk, RAM, CD-ROM

🚶 Sequential Access

You must go one by one, in order. You cannot skip. To reach item 50, you pass through items 1 to 49 first.

Like passing ball from goalkeeper → defender → midfielder → striker. The ball cannot jump from goalkeeper straight to striker. It passes through each person.

Read Write Rewind

Examples: Tape drives

📄 Text Files

A text file is words written down, stored on disk. Even though disk supports random access, text is usually read line by line.

Like reading your farm records in a notebook. To find Heap 50, you read from the first line, page by page, until you get there.

Read line Search

Example: Searching a name in a file

Section 3

Stack (LIFO): Last In, First Out

The last thing you put in is the first thing that comes out. Like stacking yams in a basket — the top yam comes out first.

🧺 What Is a Stack?

A stack is like a basket where you pile things on top of each other. You can only add to the top and remove from the top. You cannot reach the bottom without removing everything above it.

🌾 Farming Analogy

After harvest, you stack yams in a basket. The last yam you put on top is the first yam you take out. You cannot take the bottom yam without removing all the ones above it first.

⚽ Football Analogy

After training, players throw jerseys into one pile. The captain throws his last, so his jersey is on top. When collecting, captain picks first (top jersey). Coach's jersey at the bottom comes out last.

⚡ Stack Operations

Operation What It Means Example
Push Put something on top of the stack Place a yam on top of the basket
Pop Remove the top item from the stack Take the top yam from the basket
Peek Look at the top item, but do not remove it Check which yam is on top without touching it

💻 Stack in Java Code

Java
import java.util.Stack; public class StackExample { public static void main(String[] args) { Stack<String> myStack = new Stack<>(); // PUSH: Put yams on top myStack.push("Yam 1"); myStack.push("Yam 2"); myStack.push("Yam 3"); System.out.println("Size: " + myStack.size()); // POP: Remove from top, one by one while (!myStack.isEmpty()) { System.out.println(myStack.pop()); } // Output: Yam 3, Yam 2, Yam 1 (last in, first out) } }
Section 4

Queue (FIFO): First In, First Out

The first person to join the line is the first person served. Fair and orderly — like lining up for penalty kicks.

⚽ What Is a Queue?

A queue is a line. People join at the back and leave from the front. The first person who joined is the first person who leaves. Nobody cuts the line.

⚽ Football Analogy

Penalty shootout. Five players line up. Player 1 is at the front. He kicks first. Player 2 kicks second. Player 5 kicks last. The first person in line is the first to kick — fair and square.

🌾 Farming Analogy

At the grinding machine, farmers line up with their maize. First farmer there, first farmer to grind. Second farmer waits. Nobody jumps the line. Everybody takes their turn.

⚡ Queue Operations

Operation What It Means Example
Enqueue Join the back of the line New player joins penalty line at the back
Dequeue Leave from the front of the line First player kicks and leaves
Peek Look at who is at the front, but do not remove them Check who will kick next without them kicking

💻 Queue in Java Code

Java
import java.util.LinkedList; import java.util.Queue; public class QueueExample { public static void main(String[] args) { Queue<String> penaltyLine = new LinkedList<>(); // ENQUEUE: Players join the line penaltyLine.add("Player 1"); penaltyLine.add("Player 2"); penaltyLine.add("Player 3"); System.out.println("Waiting: " + penaltyLine.size()); // DEQUEUE: Players leave from the front while (!penaltyLine.isEmpty()) { System.out.println(penaltyLine.poll() + " kicks!"); } // Output: Player 1, Player 2, Player 3 (first in, first out) } }
Section 5

Linked List: A Chain of Connected Items

Each item knows where the next item is. Like players passing the ball — each player knows who to pass to next.

⛓️ What Is a Linked List?

A linked list is a chain of items called nodes. Each node holds two things: the data (the information) and a pointer (an arrow telling you where the next node is). The last node points to nothing — we call that null, meaning "the end."

⚽ Football Analogy

In a match, players pass the ball in sequence. Goalkeeper has the ball and knows to pass to defender. Defender knows to pass to midfielder. Midfielder knows to pass to striker. Striker has nobody to pass to — he shoots. Each player is a node. The ball is the data. Knowing who to pass to next is the pointer.

🌾 Farming Analogy

During planting season, you pass yam seedlings to your neighbor. Your neighbor plants his share and passes the rest to the next farmer. That farmer plants his and passes to the next. The last farmer plants his. Nobody left to pass to. End. Each farmer is a node. The seedlings are the data. Knowing which farmer is next is the pointer.

⚡ Linked List Operations

Operation What It Means Example
Insertion Add a new node to the chain A new player joins the passing sequence
Deletion Remove a node from the chain A player leaves — the one before him now passes to the one after him
Traversal Go through every node, one by one Follow the ball from goalkeeper all the way to striker
Search Find a specific node in the chain Find which player currently has the ball

import java.util.LinkedList; public class LinkedListExample { public static void main(String[] args) { LinkedList<String> players = new LinkedList<>(); players.add("Goalkeeper"); players.add("Defender"); players.add("Striker"); System.out.println("Total: " + players.size()); System.out.println("First: " + players.getFirst()); System.out.println("Last: " + players.getLast()); System.out.println("Removed: " + players.removeFirst()); for (String player : players) { System.out.println(player); } } }
Summary

Comparing All Four Structures

Here is everything side by side. Each structure has its own rule and best use case.

📦

Array

Numbered boxes in a row

Jump directly to any box using its number. Like yam heaps on your farm — walk to any heap you want.

🧺

Stack (LIFO)

Last In, First Out

Add and remove only from the top. Like stacking yams in a basket — top yam comes out first.

🚶

Queue (FIFO)

First In, First Out

Join at the back, leave from the front. Like the penalty kick line — first player kicks first.

⛓️

Linked List

Follow the chain

Each item points to the next. Like passing ball from goalkeeper to defender to midfielder to striker.
